> **主题：OS Command Injection（操作系统命令注入）**  
> **日期：2026-07-16**  
> **版本：v8alpha（故意漏洞版）vs v8beta（安全修复版）**

## 一、本次重点修复 — 命令注入漏洞

v8alpha 的 `/ping` 路由使用 `shell=True` + f-string 拼接用户输入构造系统命令，攻击者可注入任意系统命令，获取服务器完全控制权。

---

### 漏洞1：shell=True + 字符串拼接导致任意命令执行

#### 漏洞原理

v8alpha 的 `/ping` 路由（POST）将用户输入 `ip` 直接拼接到 shell 命令字符串中，并用 `shell=True` 执行。Shell 元字符（`;` `&&` `|` `` ` `` `$()`）可注入额外命令。

```python
# v8alpha：f-string 拼接 + shell=True — 致命的组合
cmd = f"ping -c 3 {ip}"
result = subprocess.check_output(cmd, shell=True, timeout=30)
```

当用户输入 `127.0.0.1; id` 时，实际执行的命令为：
```
ping -c 3 127.0.0.1; id
```
Shell 先执行 `ping`，然后执行 `id`。攻击者可注入任意系统命令。

#### 漏洞位置

| 文件 | 行号 | 说明 |
|------|------|------|
| `v8alpha/app.py` | 1499 | `cmd = f"ping -c 3 {ip}"` — f-string 拼接 |
| `v8alpha/app.py` | 1500 | `shell=True` — 启用 shell 解析 |

#### 攻击方式

| Payload | 实际执行 | 结果 |
|---------|----------|------|
| `127.0.0.1; id` | `ping -c 3 127.0.0.1; id` | 泄露当前用户 `uid=1000(kali)` |
| `127.0.0.1; cat /etc/passwd` | `ping ...; cat /etc/passwd` | 泄露系统用户数据库 |
| `127.0.0.1 && ls -la /` | `ping ... && ls -la /` | 列出根目录 |
| `8.8.8.8; whoami` | `ping ...; whoami` | 泄露执行用户身份 |

**攻击链：命令注入 → 反弹 Shell → 完全控制服务器**

#### 修复方案（v8beta）

三管齐下：

1. **`shell=False`**：使用列表传参，绕过 shell 解析
2. **正则白名单校验**：仅允许合法 IP 和域名格式
3. **HTML 自动转义**：输出不标记 `| safe`

```python
# v8beta：列表传参 + shell=False + 正则校验
def is_valid_ip_or_hostname(value):
    ipv4_re = r'^([0-9]{1,3}\.){3}[0-9]{1,3}\Z'
    hostname_re = r'^([a-zA-Z0-9]([a-zA-Z0-9\-]{0,61}[a-zA-Z0-9])?\.)*[a-zA-Z0-9]([a-zA-Z0-9\-]{0,61}[a-zA-Z0-9])?\Z'
    return bool(re.match(ipv4_re, value) or re.match(hostname_re, value))

if not is_valid_ip_or_hostname(ip):
    return render_template("ping.html", error="非法输入")

cmd = ["ping", "-c", "3", ip]  # 列表，非字符串
result = subprocess.check_output(cmd, timeout=30, stderr=subprocess.STDOUT)
```

#### 修复代码对比

| 关键点 | v8alpha（有漏洞） | v8beta（已修复） |
|--------|-------------------|------------------|
| 命令构建方式 | f-string 拼接 | 列表 `["ping", "-c", "3", ip]` |
| shell 参数 | `shell=True` | 默认 `shell=False` |
| 输入校验 | 无 | 正则白名单（IPv4+IPv6+域名） |
| 输出转义 | 无 | Jinja2 自动 HTML 转义 |

#### 效果验证

| Payload | v8alpha | v8beta |
|---------|:--:|:--:|
| `127.0.0.1` | ✅ 正常 ping | ✅ 正常 ping |
| `8.8.8.8` | ✅ 正常 | ✅ 正常 |
| `127.0.0.1; id` | 🔴 **命令执行！** | ✅ "非法输入" |
| `127.0.0.1 && whoami` | 🔴 **命令执行！** | ✅ "非法输入" |

| `$(whoami)` | 🔴 **命令执行！** | ✅ "非法输入" |
| `` `id` `` | 🔴 **命令执行！** | ✅ "非法输入" |

## 二、白盒审计同步发现并修复的问题

### 2.1 Unicode 数字绕过正则

**问题：** 原 IPv4 正则使用 `\d`，匹配 Unicode 十进制数字（如阿拉伯-印度数字 `٠`），可绕过校验。

**修复：** 将 `\d` 替换为 `[0-9]`，仅匹配 ASCII 数字。

### 2.2 错误消息泄露系统信息

**问题：** `f"Ping 执行错误: {e}"` 可能泄露 `OSError` 详情、内部路径等。

**修复：** 改为静态字符串 `"Ping 执行错误"`。

### 2.3 正则 `$` 锚点被换行绕过

**问题：** Python 的 `$` 匹配换行符前的位置。`127.0.0.1\n` 会通过校验。

**修复：** 全部正则中的 `$` 替换为 `\Z`（绝对字符串结尾）。
