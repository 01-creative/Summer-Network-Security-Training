## 漏洞点：

### 一、明文注释泄漏（信息泄露）

**漏洞位置：** `templates/login.html` 第1行

```html
<!-- 调试信息 - 默认管理员账号 用户名: admin 密码: admin123 -->
```

**漏洞描述：**
登录页面的 HTML 源代码中包含一条调试注释，直接暴露了默认管理员账号 `admin` 和密码 `admin123`。任何能够访问登录页面的用户（即使未登录），只需在浏览器中右键"查看网页源代码"即可获取管理员凭据。

**危害等级：** 🔴 高危

**攻击场景：**
1. 攻击者访问 `http://target:5000/login`
2. 右键 → 查看网页源代码
3. 第一行即可看到管理员账号密码
4. 使用泄露的凭据直接登录，获取管理员权限，查看所有用户信息

**影响范围：**
- 管理员账号完全沦陷
- 可查看任意用户敏感信息（密码、手机号、邮箱、余额）
- 结合 session 伪造漏洞可持久化控制

### 二、弱密码（认证绕过）

**漏洞位置：** `app.py` 第7-21行 USERS 字典

```python
USERS = {
    "admin": {
        "password": "admin123",     # ← 极弱密码
        ...
    },
    "alice": {
        "password": "alice2025",    # ← 弱密码（用户名+年份）
        ...
    },
}
```

**漏洞描述：**
系统内两个用户均使用弱密码：
- `admin` → `admin123`：典型的"常见弱密码TOP10"，任何字典攻击首轮即可命中
- `alice` → `alice2025`：用户名+年份的组合模式，社会工程学攻击极易猜测

**危害等级：** 🔴 高危

**攻击场景：**
1. 攻击者使用 Burp Suite / Hydra 对登录接口进行暴力破解
2. `admin123` 在常见密码字典中排名极高，秒级破解
3. 攻击者已知用户名为 `admin`（从注释泄露+枚举），直接尝试 `admin/admin123` 即可登入

**影响范围：**
- 攻击者可暴力破解任意用户密码
- 无任何登录失败限制（无验证码、无频率限制、无账户锁定）

### 三、Session 伪造（Secret Key 硬编码）

**漏洞位置：** `app.py` 第4行

```python
app.secret_key = "dev-key-2025"
```

**漏洞描述：**
Flask 的 session 机制依赖于 `secret_key` 对 cookie 进行签名。该应用的 `secret_key` 被硬编码为极易猜测的字符串 `"dev-key-2025"`，且直接暴露在源代码中。攻击者一旦获知此密钥，即可在本地伪造任意用户的 Flask session cookie。

**危害等级：** 🔴 严重

**攻击原理：**
Flask session cookie 格式为 `base64(payload).timestamp.signature`，其中签名使用 `itsdangerous` 库通过 `secret_key` 生成。掌握 `secret_key` 即可：
1. 在本地编写脚本，使用 `"dev-key-2025"` 签名一个 `{"username": "admin"}` 的 session
2. 将生成的 cookie 设置到浏览器中
3. 刷新页面即可以 admin 身份登录，无需任何密码

**攻击脚本示例（仅需3行 Python）：**
```python
from flask import Flask
from itsdangerous import URLSafeTimedSerializer
app = Flask(__name__)
app.secret_key = "dev-key-2025"
# 使用 flask.sessions 的签名机制伪造 admin session
from flask.sessions import SecureCookieSessionInterface
si = SecureCookieSessionInterface()
serializer = si.get_signing_serializer(app)
cookie = serializer.dumps({"username": "admin"})
print(cookie)  # 将此 cookie 设置到浏览器即可伪造 admin 登录
```

**危害等级：** 🔴 严重

**影响范围：**
- 无需任何密码即可伪造任意用户身份
- 攻击完全无痕（不需要访问登录接口，无日志记录）
- 可伪造不存在的用户名，可能导致权限升级

### 四、明文密码存储与前端泄露

**漏洞位置：** `app.py` 第7-21行、第30行、第43行；`templates/index.html` 第10行

```python
# app.py - 密码明文存储
USERS = {
    "admin": {
        "password": "admin123",   # ← 明文存储
        ...
    },
}

# app.py 第30行 - 将密码传递给模板
user = dict(USERS[username])
user["username"] = username
return render_template("index.html", username=username, user=user)
```

```html
<!-- templates/index.html 第10行 - 密码直接显示在页面上 -->
<li><strong>密码：</strong>{{ user.password }}</li>
```

**漏洞描述：**
1. **存储层**：密码以明文形式直接写在代码字典中，任何有代码访问权限的人都能看到所有用户密码
2. **传输层**：登录成功后，密码随用户对象传给模板
3. **展示层**：密码直接显示在首页 HTML 中，任何能物理接触已登录电脑的人（或 XSS 攻击者）都能看到密码

**危害等级：** 🔴 严重

**影响范围：**
- 管理员可看到所有用户的明文密码
- 密码在浏览器端明文显示（页面可见、DOM 可见）
- 代码泄露 → 所有用户密码泄露（一锅端）
- 违反密码安全最佳实践（OWASP A2: Cryptographic Failures）

### 五、Debug 模式开启（任意代码执行）

**漏洞位置：** `app.py` 第58行

```python
app.run(debug=True, host="0.0.0.0", port=5000)
```

**漏洞描述：**
Flask 在生产环境中开启了 `debug=True` 模式。该模式下：
1. **Werkzeug Debugger PIN**：访问任意错误页面时，将鼠标悬停在代码行上，可调出 Python 交互式调试控制台，能在服务器上执行任意 Python 代码
2. **代码热重载**：文件修改即重载，攻击者如果能写文件即可植入后门
3. **详细错误信息**：500 错误会暴露完整的堆栈跟踪、变量值和代码上下文，极大便利攻击者信息收集

**危害等级：** 🔴 严重

**攻击场景：**
1. 攻击者故意触发异常（如访问不存在的路由、传入恶意参数）
2. 错误页面显示 Werkzeug Debugger 界面
3. 攻击者点击代码行旁的终端图标，获取 Python Shell
4. 在服务器上执行 `import os; os.system("whoami")` 等任意命令

**Wekzeug Debugger PIN 爆破：**
Werkzeug debugger 需要一个 PIN 码访问控制台，但该 PIN 可通过以下信息计算得到：
- 运行用户（`getpass.getuser()`）
- Flask 模块路径
- MAC 地址
- 机器 ID（`/etc/machine-id` 或注册表）

攻击者若获得上述信息（如通过其他信息泄露），可复现 PIN 计算过程直接获取控制台访问权限。

**影响范围：**
- 可执行任意系统命令
- 服务器完全沦陷（RCE → 提权 → 横向移动）
- 可读取服务器上所有文件（含其他应用的配置和数据库密码）

### 六、绑定 0.0.0.0（过度网络暴露）

**漏洞位置：** `app.py` 第58行

```python
app.run(debug=True, host="0.0.0.0", port=5000)
```

**漏洞描述：**
应用绑定在 `0.0.0.0` 上，意味着监听所有网络接口。同一局域网（甚至公网，如果防火墙允许）内的任何设备都可以访问该服务。结合 `debug=True` 模式，等于将带调试控制台的应用直接暴露在网络上。

**危害等级：** 🟡 中危

**影响范围：**
- 局域网内任意主机可访问
- 结合 debug=True 产生严重远程攻击面
- 应绑定 `127.0.0.1` 仅允许本地访问，或通过反向代理对外暴露

---

### 七、无登录频率限制与账户锁定

**漏洞位置：** `app.py` 第34-48行 login() 函数

**漏洞描述：**
登录接口没有任何频率限制、验证码或失败次数限制。攻击者可以无限次尝试不同的用户名/密码组合，对系统进行暴力破解攻击。

**危害等级：** 🟡 中危

**影响范围：**
- 可无限次尝试登录
- 配合弱密码可快速攻破账户
- 无任何审计日志
